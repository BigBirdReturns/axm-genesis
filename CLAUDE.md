# axm-genesis — notes for Claude sessions

This is the frozen kernel and the family's doctrine shelf. Read
`README.md` and `ADOPTING.md` for the kernel; read **`docs/CONTINUITY.md`
first for anything touching the wider family** — it is the succession
document: the laws, how to amend them, the operating doctrine
(driver/hands, use-all-of-the-buffalo, the verification bar), the
roadmap, and the map of which repo owns what.

## Ground rules

- **The kernel is frozen.** Spec, digest, and signature semantics do not
  change casually; that is the entire point of this repo. Doc work here
  is common; kernel work is rare and owner-gated.
- **Doctrine docs are normative, not descriptive** — they are written so
  other repos and future sessions replicate the pattern instead of
  rediscovering it. The register is `ADOPTING.md`'s: plain declarative
  laws, honest costs, no marketing voice. Current doctrine set:
  `docs/LOCALIZATION.md` (the grammar rule), `docs/CARTRIDGE_LIFECYCLE.md`
  (the eight-stage pipeline), `docs/CONTINUITY.md` (everything else).
- **Amending a law** = RFC in the owning repo + owner acceptance + update
  the law's home doc AND `docs/CONTINUITY.md`'s index in the same PR.
  Never silently.
